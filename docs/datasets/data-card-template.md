# Dataset Card Template

Complete one private card per source and one public-safe card per derived manifest.

## Identity

- Source ID, owner, official URL, version, archive checksum
- License/terms URL, terms snapshot digest/date, allowed use, redistribution status, attribution
- Data steward, legal approver, security approver, renewal/revalidation date

## Privacy and provenance

- PII/biometric classes, consent/legal basis, jurisdictions, collection method
- Retention, revocation/deletion mechanism, enclave/storage classification
- Known risks, prohibited uses, access controls

## Composition and annotations

- Counts by modality, document/template/capture family, language/script, and task label
- Annotation schema, annotator instructions, quality checks, agreement, adjudication
- Source and derived manifest IDs; transformations and generator versions

## Splits and evaluation

- Split algorithm, leakage checks, locked-set owner
- Intended tasks, excluded tasks, baseline metrics/slices, known failure modes
